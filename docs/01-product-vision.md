## 一、使命宣言

> **让全球任何网络角落的服务，都能被安全、即时、零配置地访问。**

OmniTun 是面向开发者和企业的全栈内网穿透与私有网络平台。我们的目标是在 3 年内成为内网穿透品类的全球第一品牌。

---

## 二、我们要做什么

### 2.1 产品定义

OmniTun 是一个**企业级多租户 SaaS 平台**，提供以下核心能力：

1. **即时公网暴露**：一行命令，将本地服务映射到公网域名，自动签发 TLS 证书。
2. **全协议代理**：不仅支持 HTTP/gRPC/WebSocket，还原生支持 TCP、UDP、ICMP、SSH、RDP 等任意 L3-L7 协议。
3. **私有 Mesh 组网**：多个内网节点自动组建 WireGuard 级加密虚拟网络，支持 P2P 直连与中继回退。
4. **企业级治理**：RBAC 权限模型、SSO 集成、全量审计日志、会话录制与回放。
5. **全球边缘加速**：自建多区域 Relay 节点网络，智能就近接入，自动故障转移。

### 2.2 产品边界（我们不做的事）

| 不做 | 原因 |
|------|------|
| CDN / 静态资源托管 | 不是我们的赛道，应集成而非自建 |
| 域名注册商 | 保持中立，对接 Let's Encrypt / 自有证书 |
| VPN 替代品 | 我们是应用层 / L4 穿透，不替代全流量 VPN |
| 裸金属服务器代理 | 非目标市场 |
| P2P 文件传输 | 非核心用例，保持聚焦 |

### 2.3 一句话定位

> **OmniTun = ngrok 的易用性 + Cloudflare Tunnel 的安全性 + Tailscale 的 P2P 能力，三者合一的 SaaS 平台。**

## 三、为什么我们要做（Why Now）

### 3.1 市场结构性机会

| 趋势 | 影响 | OmniTun 的机会 |
|------|------|----------------|
| **远程办公常态化** | 全球 58% 的知识工作者至少部分远程 | 企业内部服务暴露需求爆炸式增长 |
| **微服务 / K8s 普及** | 90% 企业采用容器化，服务数量指数增长 | 开发调试场景需要轻量级服务暴露工具 |
| **IoT 爆发** | 到 2030 年全球 300 亿 IoT 设备 | 设备穿透、OTA 升级、远程运维的刚需 |
| **零信任安全成为标配** | Gartner 预测 2027 年 60% 企业采用零信任 | 传统 VPN 被淘汰，需要新一代访问方案 |
| **开发者工具市场膨胀** | 全球开发者 2030 年预计达 4500 万 | 开发者工具 SaaS 市场 CAGR 22% |
| **中国 SaaS 出海浪潮** | 中国 SaaS 加速出海，需要全球网络基础设施 | 国内友商能力薄弱，存在窗口期 |

### 3.2 竞争对手的软肋

| 竞品 | 核心缺陷 | OmniTun 的对策 |
|------|----------|----------------|
| **ngrok** | 定价昂贵、UDP/P2P 锁死在最高价、封闭生态 | 全功能 Pro 级开放，更低价格 |
| **Cloudflare Tunnel** | 深度绑定 Cloudflare CDN、国内部署困难、UDP 支持有限 | 协议中立、多区域独立部署 |
| **Tailscale** | 不提供公开 URL 暴露（Funnel 功能有限且 beta）、控制面不开源 | 公开暴露+私有组网一体 |
| **frp** | 无多租户、无 Web Dashboard、安全模型粗糙、运维门槛高 | SaaS 化简化运维、企业级治理 |
| **花生壳/NATAPP** | 功能单一、性能差、无 Mesh、无 P2P | 全功能下一代产品 |

### 3.3 窗口期判断

当前是进入该市场的最佳时间窗口：

- **2024-2026**：远程办公 + 零信任 + IoT 三重趋势叠加，需求从"有了就行"升级到"要好用、要安全"
- 头部竞品正陷入"功能锁定高价收费"与"被大厂生态绑定"的两难，创新速度放缓
- 开源替代品（frp/nps/zrok）无法满足企业需求，SaaS 化改造存在技术门槛
- AI/Agent 时代的到来让"服务暴露"从人的偶尔需求变成机器的频繁需求

## 四、如何做（How）

### 4.1 核心战略

```
OmniTun 战略三角
     ┌──────────────┐
     │   开发者优先   │
     └──────┬───────┘
            │
    ┌───────┴───────┐
    │ 开源驱动商业   │
    │  (Open Core)  │
    └───────┬───────┘
            │
    ┌───────┴───────┐
    │ 全球分布式网络  │
    └──────────────┘
```

#### 战略一：开发者优先（Developer First）

- **极致 CLI 体验**：`omnitun http 8080` 即得公网 URL + 自动 TLS
- **IDE 深度集成**：VSCode / JetBrains 插件，IDE 内一键暴露服务
- **框架原生集成**：Next.js / Vite / Spring Boot / Django 插件
- **API 优先设计**：所有功能先有 API 再有 UI，保证可编排

#### 战略二：开源驱动商业（Open Core）

- **核心隧道引擎开源**（MIT 协议）：获取开发者信任 + 社区贡献
- **企业功能闭源**：RBAC、SSO、审计、高可用中继、私有部署
- **开源即获客**：GitHub Star → 个人用户 → 团队付费 → 企业签约

#### 战略三：全球分布式网络

- **自建 Relay 节点**：亚太（新加坡/东京/香港）、北美（弗吉尼亚/旧金山）、欧洲（法兰克福/伦敦）
- **Anycast + GeoDNS**：全球就近接入，延迟 <50ms
- **P2P 智能降级**：打洞成功 → 直连（零延迟）→ 失败 → 就近中继（低延迟）→ 失败 → 任意中继（兜底）

### 4.2 商业模式

| 层级 | 月费（USD） | 核心权益 | 目标用户 |
|------|------------|----------|----------|
| **Free** | $0 | 1条活跃隧道、随机子域名、1GB/月、P2P直连、社区支持 | 个人开发者 |
| **Pro** | $12/月 | 10条隧道、3个自定义域名、100GB/月、UDP、基本访问控制、邮件支持 | 自由职业者 |
| **Team** | $49/月（最多5人） | 50条隧道、无限域名、SSO、RBAC、审计日志、Mesh组网、优先支持、API | 中小团队 |
| **Business** | $199/月 | 无限隧道、全局边缘加速、SLA 99.9%、私有Realy节点、会话录制 | 成长型企业 |
| **Enterprise** | 按需报价 | 私有部署、专属节点、SSO对接、合规报告、7×24技术支持 | 大型企业 |

**定价策略核心**：
- Pro 级即开放 UDP + P2P + 自定义域名（ngrok 需要 $20/月才给），这是获客利器
- Free 层功能完整但限制并发数，不是阉割版，是"越用越付费"的 PLG 漏斗
- 预留中国区定价（¥49/¥199/¥799），汇率联动

## 五、核心差异化总结

OmniTun 与竞品的本质差异不是"多几个功能"，而是**架构层面的代际差异**：

| 维度 | 上一代产品 | OmniTun |
|------|-----------|---------|
| 协议支持 | TCP/HTTP | 全协议（TCP/UDP/HTTP/gRPC/WS/ICMP...） |
| 拓扑模式 | 仅中继 | 中继/P2P/Mesh/级联，自适应切换 |
| 租户模型 | 单用户或无 | 原生多租户 RBAC |
| 部署模式 | SaaS 或 自建 | SaaS + 私有部署 + 混合云 |
| 边缘网络 | 无或单区域 | 全球分布式就近接入 |
| 安全模型 | 基础 TLS | 零信任 / mTLS / 持续验证 |
