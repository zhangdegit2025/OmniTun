## 一、收入仪表板 (Revenue Analytics)

### 1.1 MRR/ARR 趋势

| 属性 | 说明 |
|------|------|
| **指标** | MRR (Monthly Recurring Revenue), ARR (Annual Run Rate) |
| **粒度** | 按日 / 周 / 月 / 季度 |
| **可视化** | 折线图，含趋势线和同比/环比增长百分比 |
| **维度** | 可按计划 (Free/Pro/Team/Business/Enterprise) 拆分 |
| **预测** | 基于历史趋势的 AI 预测线（30/60/90 天） |

### 1.2 MRR 分解

| 分解维度 | 说明 | 计算方式 |
|----------|------|----------|
| **New MRR** | 本月新付费客户带来的 MRR | ∑ 新付费客户月费 |
| **Expansion MRR** | 老客户升级带来的 MRR 增长 | ∑ (新套餐月费 - 旧套餐月费) |
| **Contraction MRR** | 老客户降级带来的 MRR 减少 | ∑ (旧套餐月费 - 新套餐月费) |
| **Churn MRR** | 取消订阅导致的 MRR 流失 | ∑ 取消客户的原月费 |
| **Net New MRR** | New + Expansion - Contraction - Churn | 净增长核心指标 |

**可视化**：瀑布图，展示每月 MRR 从期初到期末的变化路径。

### 1.3 付费转化率

| 属性 | 说明 |
|------|------|
| **漏斗阶段** | 注册 → 激活 (创建首个隧道) → 试用 → 付费 |
| **关键指标** | 各阶段转化率、平均转化周期 |
| **分群** | 按获客渠道、地域、注册日期分群对比 |
| **可视化** | 漏斗图 + 趋势折线图 |

### 1.4 客户生命周期价值 (LTV)

| 属性 | 说明 |
|------|------|
| **计算方式** | LTV = ARPU × Gross Margin ÷ Monthly Churn Rate |
| **分群维度** | 按计划 / 获客渠道 / 行业 / 地域 |
| **对比** | LTV / CAC Ratio (健康标准 > 3:1) |
| **趋势** | LTV 随时间变化趋势 |

### 1.5 客户获取成本 (CAC)

| 属性 | 说明 |
|------|------|
| **计算方式** | CAC = (总营销费 + 总销售费) ÷ 新客户数 |
| **分群** | 按渠道 / 计划 / 时间周期分解 CAC |
| **CAC Payback** | CAC 回收周期 = CAC ÷ ARPU（月） |
| **可视化** | 柱状图对比各渠道 CAC |

### 1.6 Churn Rate

| 属性 | 说明 |
|------|------|
| **计算方式** | Churn Rate = 当月取消客户数 ÷ 月初总客户数 |
| **分群** | 按计划 / 客户规模 / 使用时长分群 |
| **类型** | Logo Churn (客户流失率) + Revenue Churn (收入流失率) |
| **预警** | Churn Rate 超过阈值自动告警 |

### 1.7 收入预测

| 属性 | 说明 |
|------|------|
| **预测维度** | 30 天 / 60 天 / 90 天 |
| **模型** | 基于历史 MRR 趋势、当前 Pipeline、季节性因素 |
| **置信区间** | 80% / 95% 置信区间 |
| **对比** | 实际 vs 预测对比，追踪预测准确率 |

## 二、客户管理 (CRM)

### 2.1 客户列表

| 属性 | 说明 |
|------|------|
| **列表字段** | 组织名、计划、MRR、状态 (Active/Trialing/Past Due/Cancelled)、健康评分、创建时间、最近活跃 |
| **筛选** | 按计划、状态、MRR 范围、健康评分、创建日期、地域 |
| **搜索** | 全文搜索（组织名、域名、联系人邮箱） |
| **排序** | 按任意字段排序 |
| **批量操作** | 导出 CSV、批量更新标签 |

### 2.2 客户 360 视图

单一页面汇总客户的全部信息：

| 信息域 | 内容 |
|--------|------|
| **基本信息** | 组织名、Logo、创建时间、计划、MRR、币种 |
| **联系人** | 关联的用户列表（角色、最后登录、邮箱、电话），可增删改 |
| **账单与合同** | 合同列表、发票历史、付款方式、下次付款日期 |
| **工单** | 关联的 Support Tickets 列表和状态 |
| **用量** | 活跃隧道数、数据传输量、MAU、P2P 连接数 |
| **健康评分** | 健康趋势图、风险因子列表 |
| **活动时间线** | 关键事件时间线：注册、激活、首次付费、升级、续约 |
| **备注** | 内部备注（销售/CSM 使用，客户不可见） |
| **标签** | 自定义标签（如 "VIP", "需要关注", "Churn Risk"） |

### 2.3 客户健康评分

| 属性 | 说明 |
|------|------|
| **评分模型** | 加权多因子评分：使用频率(30%) + 活跃用户增长(20%) + 功能使用多样性(15%) + 工单频率(10%) + 登录频率(10%) + 账单健康(15%) |
| **评分区间** | 🟢 Green (80-100): Healthy | 🟡 Yellow (50-79): At Risk | 🔴 Red (0-49): Critical |
| **阈值可配** | 各因子的权重和阈值可在后台调整 |
| **趋势** | 记录评分历史，展示评分变化趋势 |
| **触发** | 评分降至 Yellow/Red 时自动告警 CSM |

### 2.4 客户分段

| 属性 | 说明 |
|------|------|
| **分段维度** | 计划 (Free/Pro/Team/Business/Enterprise)、使用量级 (轻/中/重/超大)、行业、地域、获客渠道 |
| **动态分段** | 基于规则自动将客户归入分段（如 MRR > $1000 自动 "Mid-Market" 标签） |
| **分段分析** | 对比各分段的 Churn Rate, LTV, Feature Adoption |
| **导出** | 分段客户列表导出 |

## 三、账单与发票

### 3.1 发票列表

| 属性 | 说明 |
|------|------|
| **字段** | 发票号、客户、金额、币种、状态、创建日期、到期日、付款日期 |
| **筛选** | 按状态、日期范围、金额范围、客户、计划 |
| **搜索** | 按发票号、客户名搜索 |
| **导出** | CSV / PDF 批量导出 |
| **详情** | 单击查看发票详情（项目明细、税费、付款记录） |

### 3.2 发票状态

| 状态 | 说明 | 触发操作 |
|------|------|----------|
| **Pending** | 已生成，等待付款 | 客户可见，可在线支付 |
| **Paid** | 已全额支付 | 自动发送收据 |
| **Partially Paid** | 部分支付 | 显示余额 |
| **Overdue** | 已逾期 | 触发催收流程 |
| **Void** | 已作废 | 不可付款，保留记录 |
| **Refunded** | 已退款 | 显示退款金额和日期 |

### 3.3 自动催收 (Dunning)

| 属性 | 说明 |
|------|------|
| **Day -3** | 付款提醒（即将到期的发票） |
| **Day 0** | 付款失败通知 + 重试引导 |
| **Day +1/3/5** | 自动重试扣款（最多 4 次） |
| **Day +7** | 逾期提醒邮件 — 附带更新支付方式的引导 |
| **Day +14** | 第二次逾期提醒 — 告知服务暂停风险 |
| **Day +28** | 最后通知 — 服务即将暂停 |
| **Day +30** | 自动暂停服务（组织进入 Suspended 状态） |

**可配置项**：通知频率、通知文案、是否启用自动暂停。

### 3.4 手动操作

| 操作 | 说明 | 权限要求 |
|------|------|----------|
| **创建发票** | 手动开票（一次性费用、自定义金额、备注） | 运营 Admin |
| **作废发票** | 将未付款发票标记为 Void | 运营 Admin |
| **全额退款** | 退款已付款发票的全部金额 | 运营 Admin |
| **部分退款** | 退款指定金额 | 运营 Admin |
| **调整金额** | 调整未付款发票的金额（如优惠后补录） | 运营 Admin |
| **重新发送** | 重新发送发票邮件给客户 | 运营 / CSM |

所有手动操作均记录审计日志。

### 3.5 税务配置

| 属性 | 说明 |
|------|------|
| **支持税种** | VAT (欧盟), GST (澳大利亚/新西兰/新加坡/印度), Sales Tax (美国各州) |
| **税率计算** | 根据客户 Billing Address 自动匹配税率 |
| **Tax ID** | 客户可输入 VAT ID / ABN 等，系统自动验证有效性 (VIES / 各国接口) |
| **税务报告** | 按税种/地区生成税务汇总报告 |
| **免税** | 支持标记免稅組織（上传免稅证明） |

## 四、定价策略管理

### 4.1 套餐配置

可随时调整各套餐的定价和配额：

| 可配置项 | 说明 |
|----------|------|
| **价格** | 月付 / 年付价格，支持多币种定价 |
| **隧道数上限** | 每套餐可创建的隧道数量上限 |
| **数据传输量** | 每月数据传输配额 |
| **自定义域名数** | 可绑定的自定义域名数量 |
| **团队成员数** | 组织成员上限 |
| **P2P 网络数** | 可创建的 P2P 网络数量 |
| **其他配额** | Audit Log 保留期、API Rate Limit、Relay 节点可选数量 |
| **发布方式** | 修改后即时生效（仅对新订阅）/ 定时生效 |

**关键约束**：已订阅客户不受套餐变更影响，直至下次续约。

### 4.2 折扣码

| 属性 | 说明 |
|------|------|
| **折扣类型** | 百分比折扣 (如 20% off) / 固定金额折扣 (如 $10 off) |
| **适用范围** | 首月/前 N 个月/永久 |
| **次数限制** | 总使用次数上限 / 每用户限用一次 |
| **有效期** | 起始日期和截止日期 |
| **适用计划** | 指定哪些套餐可使用该折扣码 |
| **追踪** | 折扣码使用次数、带来的 MRR 折损 |
| **自定义** | 支持一次性的自定义折扣链接 |

### 4.3 试用期

| 属性 | 说明 |
|------|------|
| **期限选项** | 7 天 / 14 天 / 30 天 |
| **默认** | Pro 和 Team 新订阅默认 14 天试用 |
| **扩展** | 运营可手动延长特定客户的试用期 |
| **试用到期** | 到期前 3 天/1 天发提醒，到期后降回 Free 或要求绑定支付方式 |
| **试用期扣量** | 试用期间用量仍受套餐配额限制 |

### 4.4 自定义报价 (Enterprise)

| 属性 | 说明 |
|------|------|
| **场景** | 大客户定制化定价（超配额、多组织、私有部署） |
| **创建** | 运营后台创建自定义报价：输入各项配额和价格 |
| **有效期** | 报价有效期（如 30 天），过期自动失效 |
| **接受** | 客户通过专属链接查看并接受报价 |
| **合同** | 报价接受后自动生成合同（带电子签名流程） |

### 4.5 年度折扣

| 属性 | 说明 |
|------|------|
| **折扣率** | 年度预付享 20% 折扣 |
| **可配置** | 折扣率可在后台调整（如调整至 17% 或 25%） |
| **展示** | 定价页面同时显示月付和年付价格对比 |
| **中途升级** | 客户从月付切换年付，按比例计算剩余时间 |

## 五、客户成功 (Customer Success)

### 5.1 活跃度追踪

| 指标 | 说明 | 计算方式 |
|------|------|----------|
| **DAU** | 日活跃用户 | 当日有登录行为的唯一用户 |
| **WAU** | 周活跃用户 | 过去 7 天有登录的唯一用户 |
| **MAU** | 月活跃用户 | 过去 30 天有登录的唯一用户 |
| **DAU/MAU** | 粘性指标 | DAU ÷ MAU（健康基准 > 20%） |
| **Session** | 平均会话时长 | 用户单次登录到登出的平均时长 |

**可视化**：按组织查看活跃度折线图。

### 5.2 功能使用率

| 功能类别 | 追踪指标 |
|----------|----------|
| **隧道** | 活跃隧道数、隧道创建/删除频率、隧道运行时长 |
| **P2P** | P2P 网络数、P2P 活跃连接数 |
| **域名** | 自定义域名绑定数、TLS 证书自动签发次数 |
| **SSO** | SSO 是否已配置、SSO 登录占比 |
| **API / SDK** | API 调用量、SDK 使用分布 |
| **团队** | 团队成员数、角色分布、邀请频率 |

**应用**：识别"功能欠使用客户"以推送教程，识别"超级用户"以制作 Case Study。

### 5.3 流失预警

基于机器学习模型：

| 预警信号 | 权重 | 说明 |
|----------|------|------|
| **用量骤降** | 高 | 活跃隧道数 / API 调用量持续下降超过 30% |
| **登录衰减** | 高 | 关键联系人超过 14 天未登录 |
| **工单激增** | 中 | 近期工单突然增多，尤其是投诉/问题类 |
| **付款失败** | 高 | 最近一笔账单付款失败 |
| **功能卸载** | 中 | 删除隧道、删除团队成员、解绑域名 |
| **竞争信号** | 低 | 创建 Export 任务（数据导出可能是迁移信号） |

**模型输出**：Churn Risk Score (0-100)，高风险自动通知 CSM。

### 5.4 NPS 调查

| 属性 | 说明 |
|------|------|
| **触发** | 注册后第 90 天展示首次 NPS 调查；后续每 180 天一次 |
| **展示位置** | Dashboard 内嵌横幅（低干扰） |
| **问题** | "Would you recommend OmniTun to a friend or colleague?" (0-10) |
| **追问** | Score ≤ 6 追问原因；Score ≥ 9 可选留下 Testimonial |
| **Dashboard** | NPS 仪表板：总 NPS、趋势、按计划/段分解、Detractor 名单 |
| **Follow-up** | Detractor (0-6) 自动创建 CSM 跟进任务 |

### 5.5 客户沟通

| 触发时机 | 内容 | 渠道 |
|----------|------|------|
| **注册后立即** | 欢迎邮件 + Quick Start Guide | Email |
| **创建首条隧道** | 庆祝邮件 + 下一步引导 | Email + In-app |
| **升级付费** | 感谢邮件 + 新功能指南 | Email |
| **续约前 30 天** | 续约提醒 + 用量回顾 | Email |
| **续约前 7 天** | 续约提醒 | Email + In-app |
| **N 天未活跃** | 回访邮件 + 新功能介绍 | Email |
| **新功能发布** | Feature Update 邮件（可退订） | Email |
| **QBR** | 季度业务回顾报告（Enterprise 专属） | Email + 会议邀请 |

## 六、营销归因

### 6.1 获客渠道分析

| 渠道 | 追踪方式 | 关键指标 |
|------|----------|----------|
| **Organic Search** | Referrer + Landing Page | 注册量、付费转化率 |
| **Paid Ads** | UTM + 广告平台回传 | CAC、ROAS |
| **Referral** | Referral Code | 注册量、病毒系数 |
| **GitHub** | Referrer (github.com) | Star → Signup 转化率 |
| **Content / SEO** | Landing Page + UTM | 内容触达数 → 注册转化 |
| **Social** | UTM + Social 平台数据 | 注册量、参与度 |
| **Events/Conference** | 专属 Code / QR Code | 注册量、会后转化 |

### 6.2 转化漏斗

```
Visit Landing Page
      │  转化率 (Visitors → Signups)
      ▼
Signup (创建账号)
      │  转化率 (Signups → Activated)
      ▼
Activated (创建首条隧道)
      │  转化率 (Activated → Trial)
      ▼
Trial Started
      │  转化率 (Trial → Paid)
      ▼
Paid
      │  转化率 (Paid → Expansion)
      ▼
Expansion (升级 / 加购)
```

**可视化**：漏斗图，各阶段按日/周/月追踪；可按渠道/地域/设备分群。

### 6.3 UTM 追踪

| 参数 | 说明 | 示例 |
|------|------|------|
| `utm_source` | 流量来源 | google, github, newsletter |
| `utm_medium` | 媒介 | cpc, email, social, referral |
| `utm_campaign` | 活动名称 | product-launch-may2026 |
| `utm_content` | 内容变体 | hero-cta-vs-footer-link |
| `utm_term` | 关键词 | tunnel-service |

**追溯链**：UTM → 注册时关联到用户账号 → 后续所有转化事件带回 UTM 信息。
