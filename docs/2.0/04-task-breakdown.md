# OmniTun 2.0 — 任务拆解与执行计划

## 总览

| 里程碑 | 内容 | 任务数 | 预估工时 |
|--------|------|--------|----------|
| M1: 隧道贯通 | Agent↔Relay 数据路径 + TLS + 域名 | 8 | 16h |
| M2: 企业安全 | SSO/OIDC + MFA + RBAC + API Key + 审计 | 8 | 12h |
| M3: P2P + Mesh | STUN/TURN 打洞 + WireGuard + 自适应拓扑 | 6 | 20h |
| M4: 可观测性 | 监控 + 告警 + 分布式追踪 + 多区域 | 5 | 10h |
| M5: 计费上线 | Stripe + 用量计量 + 计划限制 | 4 | 8h |
| M6: 投产验证 | 集成测试 + E2E + 负载测试 + 安全审计 | 5 | 10h |
| **合计** | — | **36** | **~76h** |

---

## M1: 隧道贯通（最高优先级）

### T1.1: WebSocket Gateway 集成到 server main
| 字段 | 内容 |
|------|------|
| **依赖** | 无 |
| **产出** | `cmd/server/main.go` 增加 `/gateway/agent` WebSocket 路由，复用 `internal/gateway/hub.go` |
| **验收** | Agent WebSocket 连接 localhost:8080/gateway → 握手成功 → Hub.agentCount() == 1 → 心跳正常 |
| **工时** | 2h |

### T1.2: Agent CLI 完整接入
| 字段 | 内容 |
|------|------|
| **依赖** | T1.1 |
| **产出** | `cmd/client/cmd/http.go` 真实调用 REST API 创建隧道 → 启动 → 连接 WS Gateway → 连接 Relay 数据通道 |
| **验收** | `omnitun http 8080` → Agent 建立 WS 控制连接 + QUIC 数据通道 → 隧道状态变为 active |
| **工时** | 3h |

### T1.3: Relay Token 验证 + Agent 认证
| 字段 | 内容 |
|------|------|
| **依赖** | T1.1 |
| **产出** | Relay 端 TUNNEL_CONNECT 帧的 tunnel_token 验证（HMAC-SHA256 + 过期检查） |
| **验收** | 有效 token → 建立隧道；无效 token → 拒绝连接 |
| **工时** | 1h |

### T1.4: 端到端 HTTP 穿透验证
| 字段 | 内容 |
|------|------|
| **依赖** | T1.2, T1.3 |
| **产出** | 完整的 Agent ↔ Relay ↔ 公网 HTTP 请求穿透链路 |
| **验收** | 启动本地 HTTP 服务 → `omnitun http 8080` → curl http://localhost:4443/{slug} → 返回本地服务响应 |
| **工时** | 3h |

### T1.5: ACME TLS 自动签发
| 字段 | 内容 |
|------|------|
| **依赖** | T1.4 |
| **产出** | `internal/tls/acme.go` 使用 `go-acme/lego` 库实现证书自动签发和续期 |
| **验收** | 创建隧道 → 自动获取 *.omnitun.io 泛域名证书 → Relay 可提供 HTTPS |
| **工时** | 2h |

### T1.6: 自定义域名支持
| 字段 | 内容 |
|------|------|
| **依赖** | T1.5 |
| **产出** | DNS 验证流程（CNAME + TXT）、定期检查、验证通过后签发证书 |
| **验收** | 用户添加自定义域名 → 配置 DNS → 验证通过 → HTTPS 可用 |
| **工时** | 2h |

### T1.7: 隧道实时流量日志
| 字段 | 内容 |
|------|------|
| **依赖** | T1.4（有真实流量）+ ClickHouse 容器运行 |
| **产出** | Relay 转发时记录流量事件 → 写入 ClickHouse traffic_events 表 → Dashboard 实时展示 |
| **验收** | 发起 HTTP 请求 → TunnelDetail 页面显示请求日志（IP/Method/Path/Status/Bytes） |
| **工时** | 2h |

### T1.8: Relay 高可用
| 字段 | 内容 |
|------|------|
| **依赖** | T1.4 |
| **产出** | 同区域多 Relay → Agent 断线自动切换到其他 Relay → 隧道状态保持 |
| **验收** | 手动停止当前 Relay → Agent 重连到另一 Relay → 隧道恢复 < 30s |
| **工时** | 1h |

## M3: P2P + Mesh

### T3.1: NAT 类型探测
| 字段 | 内容 |
|------|------|
| **依赖** | 无 |
| **产出** | Agent 连接 STUN 服务器 → 探测 NAT 类型 → 上报到控制面 |
| **验收** | Agent 正确识别 Full Cone / Restricted / Symmetric NAT |
| **工时** | 3h |

⚠️ 环境限制：需下载新 Go 模块。实现代码路径 + mock 验证，不要求真实外部服务连通。

### T3.2: UDP Hole Punching
| 字段 | 内容 |
|------|------|
| **依赖** | T3.1 |
| **产出** | 两个 Agent 交换端点信息 → 同时发包打洞 → 成功后直连 |
| **验收** | 非对称 NAT 环境打洞成功率 > 80% |
| **工时** | 4h |

### T3.3: TURN Relay 自建
| 字段 | 内容 |
|------|------|
| **依赖** | T3.1 |
| **产出** | 自建 TURN 服务器 → 对称 NAT 环境下通过 TURN 转发 |
| **验收** | 打洞失败 → 自动降级到 TURN 中继 → 数据正常传输 |
| **工时** | 3h |

⚠️ 环境限制：需下载新 Go 模块。实现代码路径 + mock 验证，不要求真实外部服务连通。

### T3.4: WireGuard 集成
| 字段 | 内容 |

| **依赖** | T3.2 |

| **验收** | 两个 Agent 建立 WireGuard 隧道 → ping 通 → TCP/UDP 数据转发正常 |
| **工时** | 4h |

⚠️ 环境限制：需下载新 Go 模块。实现代码路径 + mock 验证，不要求真实外部服务连通。

### T3.5: Mesh 组网 Dashboard
| 字段 | 内容 |
|------|------|
| **依赖** | T3.4 |

| **验收** | Dashboard 可创建网络 → 查看节点 → 查看拓扑图 |

### T3.6: 自适应拓扑引擎
| 字段 | 内容 |
|------|------|

                              │
